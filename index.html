Production-Grade Design Decisions

Text-to-SQL with Safety Chain

  • NL → SQL via Vanna 2.0 with schema-aware generation
  • Safety validator rejects destructive / schema-altering statements
  • LLM-as-Judge scores query for correctness & intent alignment
  • Cryptographic approval gate (token_hex) for human-in-the-loop
  • Full per-query audit log + /explain endpoint

Corrective Self-Reflective RAG

  • HyDE: 3 hypothetical document queries for expansion
  • Hybrid search: Dense + Sparse (BM25) with RRF fusion
  • Voyage AI cross-encoder reranking for precision
  • CRAG: per-chunk scoring → Tavily web fallback on low confidence
  • SRAG: factual support + usefulness verification (≤2 retries)

Document-Driven Mutations

  • Excel/CSV upload → LLM-assisted column mapping to schema
  • Auto-classify: INSERT / UPDATE / DELETE
  • Rule validation against business_rules/rules.json
  • LLM audit verifies intent matches data payload
  • Atomic PostgreSQL transaction with auto-rollback

Infrastructure & Reliability

  • 4-tier Redis cache (SQL-gen / Embeddings / Chunks / Results)
  • SHA-256 deduplication for chunks across uploads
  • LiteLLM Router: Groq ×4 keys → OpenAI fallback
  • CI/CD: Ruff + pytest → ECR → zero-downtime health-check rollback
  • Opik: full LLM trace logging per request
